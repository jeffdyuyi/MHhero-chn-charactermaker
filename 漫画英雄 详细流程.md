### 👨‍💻 开发者数据结构参考 (JSON & 逻辑示例)

在网页开发中，由于 d66 表格存在范围合并（如 1-2 对应同一个能力），建议在 JSON 中采用 `min` 和 `max` 匹配，或者直接拍平成数组下标，这样检索最快。

```javascript
// 示例：运动系 (Movement) 字典构型
const powersDictionary = {
  "movement": {
    name: "运动系",
    // subRolls 对应第一枚 d6 的分组，1-3 为组A，4-6 为组B
    subRolls: [
      {
        range: [1, 3], 
        // 数组索引 0-5 对应第二枚 d6 的 1-6
        results: ["挖掘洞穴", "位面旅行", "飞行", "飞行", "跳跃", "跳跃"] 
      },
      {
        range: [4, 6],
        results: ["高速旋转", "超高速移动", "超高速移动", "摇荡", "瞬间移动", "墙面爬行"]
      }
    ]
  }
};

// 🎲 随机生成一个运动系能力的函数示例
function generateMovementPower() {
  const d6_first = Math.floor(Math.random() * 6) + 1;  // 1~6
  const d6_second = Math.floor(Math.random() * 6) + 1; // 1~6
  
  // 匹配第一枚骰子对应的组
  const group = powersDictionary.movement.subRolls.find(
    g => d6_first >= g.range[0] && d6_first <= g.range[1]
  );
  
  // 根据第二枚骰子获取最终能力名 (索引为 d6_second - 1)
  const powerName = group.results[d6_second - 1];
  
  return powerName;
}
```

### 🧩 第三阶段 (附加)：能力调整模组 (Power Modifiers)

#### 1. 标准附带效果表 (Extras)

| :--- | :--- |

| **仅限可视范围**| 该项特殊能力只能对可视范围内的目标（或者使用者在目标的可视范围内时）造成影响。 |

| **特技禁用** | 该项特殊能力无法作为特技使用，只能发挥最基本的效果。 |
| **特殊限定** | 该项特殊能力只能在特定情况下或对抗特定目标时才能生效（如仅限夜晚、仅限愤怒时、只能对男性等）。 |
| **仅限他人** | 该项特殊能力只能对他人造成影响，无法作用于自身。 |
| **施放动作** | 使用者需要做出某些施放动作才能使用特殊能力（如咒语、手势），无法做出动作则不能使用。 |
| **预先准备** | 使用者需要至少进行一画页的准备才能触发特殊能力。 |
| **限制来源** | 使用者无法生成特殊能力所需的能量或物质，必须依赖外部来源（受限于可用来源的有效等级）。 |
| **缩短时间** | 原本持续时间为延续或永久的特殊能力效果，现在只能持续与等级相同的画页数。 |
| **耐力消耗** | 使用该项特殊能力会损耗 2 点耐力值。 |
| **不受控制** | 该项特殊能力的触发有时不遵循使用者的意愿，而是受到游戏主持人控制。 |
| **成败未知** | 使用该项特殊能力时需要进行掷骰，若结果为1点或2点，则能力不会生效。 |
| **变化莫测** | 在战斗或紧急状态使用时需要掷骰，若结果为1点或2点，则能力会随机产生效果，不遵照使用者意图。 |

### 🧮 第一部分：衍生属性计算规则 (Derived Attributes)

衍生属性由角色的基础数据动态计算得出，代表了角色在游戏中的“生命值”与“能量值”。

| 衍生属性名称 | 🧮 核心计算公式 | ⚠️ 特殊调整与限制条件 (开发必读) |
| :--- | :--- | :--- |
| ❤️ **耐力**<br>*(Stamina)* | **`耐力 = 力量等级 + 意志等级`** | 1. 这是角色的“血条”，用于承受伤害。<br>2. 随**力量(Strength)**或**意志(Willpower)**的改变而实时同步增减。<br>3. 某些特殊能力（如*拔地而起*、*浓缩密度*）如果提升了力量，也会**间接提升耐力**。 |
| ⚡ **决意**<br>*(Resolve)* | **`决意 = 6 - 特殊能力总数量`** | 1. **保底机制**：无论特殊能力有多少项，初始决意等级**最低不能低于 1**。`(Math.max(1, 6 - powerCount))`。<br>2. **高属性惩罚**：如果角色的任何一项属性（通过起源加值等方式）**达到 7 级或更高**，每有一项，最大决意点数就必须 **扣除 1 点**。<br>3. **限制条件补点**：某些特殊能力如果挂载了“减少1点决意花费”的限制条件，也会影响这里的结算。 |

### 🛡️ 第二部分：车卡核心校验逻辑 (Validation Rules)

在玩家点击“完成车卡”或“导出卡片”按钮前，系统需要校验以下规则是否满足。建议将这些逻辑写为验证函数，在 UI 上实时反馈警告信息。

#### 1. 属性总和保底校验 (Attribute Sum Check)
*《漫画英雄》为防止玩家脸太黑，设置了“重开”阈值。*
* **逻辑**：计算六项基础属性的总和（必须**包含**第一阶段“能力起源”带来的数值加成）。
* **阈值**：总和必须 **≥ 20 级**。
* **UI 动作**：如果 `< 20`，按钮置灰，并提示：*“英雄的属性等级总和低于20，建议重新掷骰或分配！”*

#### 2. 等级上限溢出校验 (Level Cap Check)
*凡人之躯比肩神明也是有极限的。*
* **逻辑**：遍历六大基础属性和所有的特殊能力等级。
* **阈值**：任何一项基础能力的等级 **不得超过 10 级**。
* **UI 动作**：如果发现 `level > 10`，提示：*“任何属性或特殊能力在初始建卡时不能超过10级！”*

#### 3. 专长投入校验 (Specialty Cap Check)
*专长的熟练度分为基础(+1)、专家(+2)、大师(+3)。*
* **逻辑**：检查玩家在单一专长上投入的点数/次数。
* **阈值**：同一项专长最多只能被选择/投入 **3 次**（即最高为“大师”级）。
* **UI 动作**：当某专长等级到达 3 时，禁用该专长的 `+` 按钮。

#### 4. 特质完整性校验 (Qualities Check)
*特质是角色扮演和获取决意点的关键文本。*
* **逻辑**：检查特质输入框的非空状态。
* **阈值**：标准开局要求玩家必须填写 **3 项** 特质（通常是身份、动机、弱点/羁绊各一）。
* **UI 动作**：如果特质数组长度 `< 3`，提示：*“请为你的英雄填写至少3项特质，以便在游戏中赚取决意点。”*

### 👨‍💻 第三部分：前端开发逻辑参考 (JavaScript / TypeScript 伪代码)

以下代码展示了如何在状态管理中实时计算衍生属性并进行车卡校验：

```typescript
// 假设这是角色的状态 Store
const heroState = {
  origin: "改造", // 来源
  attributes: {
    brawn: 3,
    coordination: 4,
    strength: 8,   // 力量 8，触发高属性惩罚
    intellect: 3,
    awareness: 4,
    willpower: 3
  },
  powers: [
    { name: "光环", level: 6 },
    { name: "飞行", level: 4 }
  ], // 2项特殊能力
  specialties: [
    { name: "武术", level: 3 },
    { name: "科学", level: 1 }
  ],
  qualities: ["失忆的超级士兵", "极度怕火"] // 只有2项，不满足校验
};

// 🧮 1. 计算衍生属性 (Computed)
function calculateDerivedStats(state) {
  // 耐力 = 力量 + 意志
  const stamina = state.attributes.strength + state.attributes.willpower;
  
  // 基础决意 = 6 - 特殊能力数量 (保底为1)
  let resolve = Math.max(1, 6 - state.powers.length);
  
  // 高属性惩罚：检查是否有属性 >= 7
  const highStatCount = Object.values(state.attributes).filter(val => val >= 7).length;
  resolve = Math.max(0, resolve - highStatCount); // 扣除惩罚，决意甚至可能扣到0

  return { stamina, resolve };
}

// 🛡️ 2. 全局校验函数 (Validation)
function validateCharacter(state) {
  const errors = [];

  // 校验属性总和 (包含起源加值后的最终总和)
  const totalAttributes = Object.values(state.attributes).reduce((a, b) => a + b, 0);
  if (totalAttributes < 20) {
    errors.push("英雄属性总和过低（低于20），建议重新分配或重掷！");
  }

  // 校验等级上限
  const allStats = [...Object.values(state.attributes), ...state.powers.map(p => p.level)];
  if (allStats.some(level => level > 10)) {
    errors.push("能力或属性等级不能超过 10 级！");
  }

  // 校验专长上限
  if (state.specialties.some(s => s.level > 3)) {
    errors.push("单一专长最高只能达到大师级（3级）！");
  }

  // 校验特质数量
  if (state.qualities.length < 3) {
    errors.push("请至少填写 3 项特质！");
  }

  return {
    isValid: errors.length === 0,
    errors
  };
}

// 执行测试
const derived = calculateDerivedStats(heroState);
console.log("衍生属性:", derived); 
// 输出: 衍生属性: { stamina: 11, resolve: 3 } 
// 解释: 决意 = 6 - 2项能力 = 4。但因为力量是8(>=7)，扣除1点，最终决意为3。

const validation = validateCharacter(heroState);
console.log("校验结果:", validation); 
// 输出: 校验结果: { isValid: false, errors: [ '请至少填写 3 项特质！' ] }
```
