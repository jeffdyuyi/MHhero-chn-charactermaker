# 漫画英雄TRPG车卡器开发文档

## 一、系统架构

### 1. 核心架构

本项目采用模块化、组件化的架构设计，主要分为以下几个核心模块：

- **引导式流程控制器**：负责管理角色创建的各个步骤，处理步骤间的导航和数据传递
- **步骤组件**：每个创建步骤的具体实现，包括模式选择、能力起源、基础属性等
- **数据验证系统**：确保所有用户输入都符合规则要求
- **错误处理系统**：捕获和处理系统错误，提供友好的错误提示
- **安全性系统**：防止XSS攻击和数据注入，保护用户数据
- **性能优化系统**：优化用户输入和渲染性能

## 二、核心模块详解

### 1. 引导式流程控制器

**文件**：`js/ui/guided-creation-flow.js`

**功能**：
- 管理角色创建的7个步骤
- 处理步骤间的导航和数据传递
- 提供角色保存、导出和删除功能
- 集成数据验证和安全性措施

**核心方法**：
- `goToStep(index)`：跳转到指定步骤
- `nextStep()`：进入下一步
- `prevStep()`：返回上一步
- `validateCurrentStep()`：验证当前步骤数据
- `saveCharacter()`：保存角色数据
- `loadCharacter(characterId)`：加载已有角色
- `deleteCharacter()`：删除角色

### 2. 步骤组件

**基础类**：`Step`
- `render()`：渲染步骤内容
- `validate()`：验证步骤数据
- `saveData()`：保存步骤数据

**具体步骤**：
- `ModeSelectionStep`：创建模式选择
- `OriginSelectionStep`：能力起源选择
- `AttributesStep`：基础属性分配
- `PowersStep`：超凡能力选择
- `SpecialtiesStep`：生活专长选择
- `CharacterInfoStep`：角色信息填写
- `PreviewStep`：角色预览和保存

### 3. 数据验证系统

**文件**：`js/core/validation.js`

**功能**：
- 验证角色完整数据
- 验证能力数据
- 验证专长数据
- 验证属性值范围
- 验证购点模式点数

### 4. 错误处理系统

**文件**：`js/core/error-handler.js`

**功能**：
- 全局错误捕获
- 友好的错误提示
- 成功和信息提示

### 5. 安全性系统

**文件**：`js/core/security.js`

**功能**：
- 防止XSS攻击
- 数据清理和验证
- 生成唯一ID
- 本地存储安全性检查

### 6. 性能优化系统

**文件**：`js/core/performance.js`

**功能**：
- `debounce()`：优化用户输入
- `throttle()`：优化频繁操作
- `memoize()`：缓存计算结果
- `lazyLoadImages()`：图片懒加载

## 三、使用方法

### 1. 角色创建流程

1. **选择创建模式**：随机模式或购点模式
2. **选择能力起源**：受训、改造、变种、外星、魔法、科技
3. **分配基础属性**：勇猛、协调、力量、智力、感知、意志
4. **选择超凡能力**：从能力库中选择并设置等级
5. **选择生活专长**：从专长库中选择并设置等级
6. **填写角色信息**：名称、身份、动机、描述
7. **预览和保存**：查看角色完整信息并保存

### 2. 角色编辑流程

1. 在已保存角色页面选择要编辑的角色
2. 系统会加载角色数据到引导式流程
3. 修改相应的信息
4. 保存修改后的角色

### 3. 角色删除流程

1. 在已保存角色页面选择要删除的角色
2. 进入编辑模式
3. 在预览步骤点击删除按钮
4. 确认删除操作

### 4. 数据导出

在预览步骤点击导出按钮，系统会生成并下载角色的JSON数据文件。

## 四、技术栈

- **前端**：原生 JavaScript + ES Modules
- **样式**：CSS3 + CSS变量
- **存储**：LocalStorage
- **可选库**：html2canvas (用于图片导出)

## 五、开发指南

### 1. 添加新步骤

1. 创建一个继承自 `Step` 的新类
2. 实现 `render()`、`validate()` 和 `saveData()` 方法
3. 在 `GuidedCreationFlow` 的 `steps` 数组中添加新步骤

### 2. 添加新能力或专长

1. 在 `data/powers.js` 或 `data/specialties.js` 中添加新数据
2. 确保数据格式符合现有结构

### 3. 性能优化

- 对频繁触发的事件使用 `debounce` 或 `throttle`
- 对复杂计算使用 `memoize` 缓存结果
- 对图片使用 `lazyLoadImages` 进行懒加载

### 4. 安全性注意事项

- 所有用户输入必须经过 `sanitizeInput` 处理
- 所有数据保存前必须经过 `sanitizeCharacterData` 清理
- 验证所有输入的合法性和范围
